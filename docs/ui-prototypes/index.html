Exam to HTML · M2 UI 方向选择

下面是 4 个可直接打开的静态 HTML 方案。它们都遵守设计文档主线：教师只看见「拖 PDF → 开始转换 → HTML 自动打开」，输出位置默认桌面，高级设置折叠，不主动推销 MinerU token。

预览方式 每张卡片点「打开全屏预览」。这些文件只在 docs/ui-prototypes 下，不改现有生产 UI。
最稳妥

01 · Drop-and-Go 极简向导

最接近设计文档 §1.1 原始 mockup：单窗口、单主按钮、三步状态条。适合先把 v1 做到「李老师 5 秒懂」。

低学习成本低实现风险适合 1366×768
打开全屏预览看大图
最专业

02 · Workbench 双栏工作台

左侧控制三步，右侧实时预览和日志，像 Windows 桌面工具。适合强调「我知道系统在做什么」。

左控右看进度可信桌面工具气质
打开全屏预览看大图
最有产品感

03 · One Desk Bento 课桌

把「输入 / 队列 / 最近进度」拆成三张卡。更像成熟小工具，支持未来多 PDF 排队，但仍不暴露技术细节。

卡片清晰可扩展队列完成记录明确
打开全屏预览看大图
最有记忆点

04 · Desk Drawer 纸张抽屉

温暖纸张隐喻：像把 PDF 放到备课桌上处理。更有品牌气质，但需要控制不要变得过度文艺。

温暖教育感纸张隐喻不幼稚
打开全屏预览看大图
你刚刚提的

02b · Workbench 暖橙版

02 的版式 + PDF2PPT 的暖橙 token：珊瑚橙 CTA、暖灰底、Inter 字体。结构和功能完全不变，只换皮。

珊瑚橙主色暖灰底PDF2PPT 同源
打开全屏预览看大图

我的初步判断：如果目标是 M2 稳健落地，优先选 01；如果想让产品一眼不像 demo，选 02；如果已经确定要支持多 PDF 队列，选 03；04 适合后续品牌化，但不是第一优先。
